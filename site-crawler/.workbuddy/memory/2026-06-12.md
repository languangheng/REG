## 修复 Windows GBK 控制台 UnicodeEncodeError (Commit: e8aa1be)
- 根因：Windows 控制台默认 GBK 编码，print() 输出 emoji（✅❌⚠️）时抛 UnicodeEncodeError，导致子进程直接崩溃
- 修复（三层防御）：
  1. group_crawler.py 顶部：sys.stdout.reconfigure(encoding='utf-8') 强制子进程 stdout 用 UTF-8
  2. _log()/_emit() 改用 _safe_print()：优先 sys.stdout.buffer.write(UTF-8字节) 绕过控制台编码，失败则降级 print + GBK replace 兜底
  3. site_manager.py 子进程 env 注入 PYTHONIOENCODING=utf-8 双重保险
- 验证：语法检查通过，group_crawler.py 内无裸 print() 调用

## CF (CloudFlare) 持久化 Session 修复 (21:53-22:05)
- 问题：每次爬取 spawn subprocess → 新浏览器 session → CF 重新触发，每次等 2.5 分钟
- 根因：subprocess.Popen 子进程创建新 BrowserActClient → 新 session → CF 重验证
- 修复内容：
  1. **site_manager.py** — 爬取从 subprocess 改为 in-thread:
     - `_get_persistent_client()` 维护全局 BrowserActClient（session="group_crawl"）
     - `_warm_session(url)` 首次预热（ensure_session+Cookie保存），后续同域名直接 navigate
     - `/api/crawl` 和 `/api/crawl/test` 改用 threading.Thread + queue.Queue SSE
     - `_crawl_client_busy` 锁防并发
  2. **engine.py** — 新增 `set_event_callback()` / `clear_event_callback()`:
     - `_log_msg()` 和 `_emit()` 当 callback 存在时推送到 queue 而非 stdout
  3. **group_crawler.py** — `_crawl_with_engine` 新增 `client` 参数:
     - 外部传入 client 时跳过 session 初始化，复用已有 session
  4. **browser_act.py** — `ensure_session()` 优化:
     - session 存活时先 12s 快速渲染检测（而非直接进入 60s CF 等待）
     - 检测到 URL-title 时强制重建 session（不再接受未渲染页面）
     - 增加 "请稍候"、"just a moment" 中文 CF 检测
- 效果验证：
  - 首次爬取: ~10 秒（Cookie 导入 + 快速 CF 验证）
  - 后续爬取: ~2 秒（域名已预热，直接 navigate，跳过 CF）
  - 对比旧架构: 每次 2.5 分钟
