## 一、项目目标

**最终目标**：实现一个**零配置视频网站爬虫生成器**，用户只需在浏览器中完成几步交互（选链接、选播放按钮），系统自动分析目标网站结构，生成 `sites.yaml` 爬虫配置文件。

**核心价值**：用户不需要懂 HTML、CSS、XPath，只需要"点点点"，就能为任意视频网站生成爬虫配置。

---

## 二、整体架构

### 2.1 预期工作流程

```
用户执行命令:
  python -m crawler.config_wizard --url <入口页URL> --name <网站名>

浏览器中发生的交互:
  ① 入口页（列表页） → 用户点一个视频链接
  ② 详情页 → 用户点播放/观影按钮
  ③ 播放页 → 系统自动提取 m3u8/mp4 等流地址
  ④ 完成 → 配置写入 sites.yaml
```

### 2.2 当前实际架构（重构后）

```
Python 层（config_wizard_refactored.py）
├── wizard_state 字典（唯一状态源）
├── run_wizard() 主循环
│   ├── 注入覆盖层（JS）
│   ├── 轮询 window.__cw_event（每2秒）
│   ├── handle_event() 更新状态
│   └── 重新注入
└── write_config() 写入 sites.yaml

BrowserActClient（browser_act.py）
└── 封装 browser-act CLI，提供 navigate / evaluate_js 等方法

browser-act CLI（外部工具）
└── 控制真实 Chrome/Edge 浏览器，执行 JS eval

JS 层（OVERLAY_JS_STATELESS，内嵌在 Python 中）
├── window.__cw_state ← Python 每次注入时传入
├── 渲染 UI（根据 state.stage）
└── 用户操作 → window.__cw_event = {type, data} → Python 轮询读取
```

### 3.2 已解决的技术问题

1. **`sys.stdout` 重定向导致 import 时文件关闭**：`browser_act.py` 开头重定向了 stdout，import 时 `print()` 导致文件提前关闭 → 已删除重定向代码

2. **中文编码乱码**：`browser-act eval --stdin` 通过 PowerShell pipe 传 UTF-8 临时文件时，GBK 解码失败 → 改用 `errors='surrogateescape'`，并最终将覆盖层语言改为英文

3. **`browser-act` 命令差异**：无 `open` 命令，需用 `navigate` 或 `browser open`

4. **JS `sessionStorage` 状态恢复失效**：页面跳转后 `window.load` 事件不触发，JS 状态丢失 → 重构为 Python 管理状态

5. **ES6 语法不兼容**：`eval` 上下文无 Babel，模板字符串反引号等不工作 → 全部改 ES5 语法

## 四、当前遇到的具体问题

### 4.1 近期测试失败（19:39 ~ 19:40）

**错误**：`SyntaxError: Unexpected identifier 'close_panel'`  
**原因**：JS 模板中 `onclick="window.__cw_event={type:'close_panel'}"` 的单引号与 CSS 中的内容冲突，浏览器解析 HTML 时提前截断

**已修复**：将 `onclick` 属性改为 `id="cw-close-btn"`，在 JS 中用 `document.getElementById('cw-close-btn').onclick = function(){...}` 绑定

### 4.2 进程启动不稳定

Webchat 中 `exec` 启动的后台进程经常在第一次 `yieldMs` 后显示 `SIGKILL` 或 `aborted`，实际进程可能已退出。  
**影响**：难以在 Webchat 中可靠观察日志
