# 项目约束文档

> **重要**：本文档定义了项目开发必须遵循的严格工作流程。任何开发工作都必须遵守这些约束。

## 📋 核心工作流：Plan → 确认 → 实现 → 测试 → 验收 → 提交

### 1. 📝 Plan（计划）阶段

**必须做到**：
- ✅ 在动手之前，先制定完整的计划
- ✅ 计划必须包含：
  - 要做什么（目标）
  - 怎么做（具体步骤）
  - 预期输出（可验证的结果）
  - 潜在风险（可能的问题）
- ✅ **计划必须让用户确认后才能执行**
- ✅ 如果用户有修改意见，必须调整计划后重新确认

**禁止**：
- ❌ 没有计划就动手写代码
- ❌ 计划没让用户确认就开始实现
- ❌ 边做边想（除非是探索性研究）

---

### 2. 💻 实现（Implementation）阶段

**必须做到**：
- ✅ 严格按照计划执行，不得擅自偏离
- ✅ **写代码之前，先想好怎么测试这段代码**
  - 测试输入是什么？
  - 预期输出是什么？
  - 如何验证代码正确性？
- ✅ **不能验证的代码，一句都不能写**
- ✅ 遇到新问题时：
  1. **立即停止**
  2. 记录问题
  3. 反馈给用户
  4. 等待用户指示（是否需要调整计划）

**禁止**：
- ❌ 写无法测试的代码
- ❌ 遇到新问题后继续盲目实现
- ❌ 跳过测试环节

---

### 3. 🧪 测试（Testing）阶段

**必须做到**：
- ✅ 实现完成后，必须进行**全面测试**
- ✅ **特别重要**：模拟真实用户操作，做**完整功能测试**
- ✅ 使用 `browser-act` 自动化操作 WebUI：
  - 打开浏览器
  - 模拟点击、输入、提交等操作
  - 验证页面响应
  - 检查功能是否正常工作
- ✅ 测试必须覆盖：
  - 正常流程
  - 边界条件
  - 错误处理

**禁止**：
- ❌ 只做单元测试，不做端到端测试
- ❌ 假设功能正常，不实际验证
- ❌ 让用户手动测试（AI 应该自己完成测试）

---

### 4. ✅ 验收（Acceptance）阶段

**必须做到**：
- ✅ 测试通过后，向用户展示测试结果
- ✅ 提供清晰的测试报告：
  - 测试了什么
  - 测试通过/失败
  - 如果有失败，原因是什么
- ✅ **等待用户确认后，才能提交代码**

**禁止**：
- ❌ 测试还没做就提交代码
- ❌ 测试失败了还提交代码

---

### 5. 📦 提交（Commit）阶段

**必须做到**：
- ✅ 代码通过验收后，才能提交
- ✅ 提交信息要清晰：
  - 做了什么改动
  - 为什么做这些改动
  - 解决了什么问题

## 📁 文件组织规范

**核心原则**：保持主项目干净，所有非核心文件必须放在指定目录。

### 目录结构

```
site-crawler/
├── PROJECT_CONSTRAINTS.md   # 约束文档（唯一保留在根目录的文档）
├── start.py                  # 启动脚本（核心功能）
├── run.bat                  # 启动脚本（核心功能）
├── site_manager.py           # 主程序（核心功能）
├── requirements.txt          # 依赖列表（核心功能）
├── sites.json               # 站点配置（核心功能）
├── crawler/                 # 核心代码目录
├── templates/              # 前端模板目录
├── tmp/                    # 临时文件目录
│   ├── doc/               # 旧文档（不再需要）
│   ├── old/               # 废弃文件
│   └── ...               # 测试文件、临时文件
├── log/                    # 日志目录（每次运行独立子目录）
│   ├── 20260612_091834/   # 第1次运行日志
│   │   ├── all.log
│   │   └── err.log
│   └── 20260612_143012/   # 最新运行日志（排错首选）
│       ├── all.log
│       └── err.log
├── out/                    # 程序输出目录（爬取结果等）
├── doc/                    # 新文档目录（以后生成的文档）
└── .gitignore             # Git 配置
```

### 文件放置规则

| 文件类型 | 放置目录 | 示例 |
|----------|----------|------|
| 核心程序文件 | 根目录 | `site_manager.py`, `start.py` |
| 核心代码目录 | 根目录 | `crawler/`, `templates/` |
| 配置文件 | 根目录 | `sites.json`, `requirements.txt` |
| 测试文件 | `tmp/` | `test_*.py`, `auto_test_*.py` |
| 临时文件 | `tmp/` | `service.log`, `*.tmp` |
| 废弃文件 | `tmp/old/` | 旧版本脚本、不再使用的代码 |
| 旧文档 | `tmp/doc/` | 历史文档、不再需要的说明 |
| 新文档 | `doc/` | 以后生成的项目文档 |
| 程序输出 | `out/` | 爬取结果、生成的数据文件 |
| 日志文件 | `log/<时间戳>/` | 每次运行自动创建目录，排错看最新目录 |

### 严格约束

**必须做到**：
- ✅ 生成测试文件时，必须放在 `tmp/` 目录
- ✅ 程序临时生成的产物，必须放在 `out/` 目录
- ✅ 创建文档时，必须放在 `doc/` 目录
- ✅ 根目录只保留核心程序文件和 `PROJECT_CONSTRAINTS.md`
- ✅ 清理时，将非核心文件移动到 `tmp/` 对应目录

**禁止**：
- ❌ 在根目录生成测试文件
- ❌ 在根目录留下临时文件
- ❌ 在根目录放置文档（除了 `PROJECT_CONSTRAINTS.md`）
- ❌ 将废弃文件直接删除（必须移动到 `tmp/old/`）

## 📝 日志规范（最高优先级）

**核心原则**：排错只有一处地方查看，就是日志文件。不要到处找日志。

### 日志目录结构

每次程序启动时，自动在 `log/` 下创建以时间戳命名的独立目录：

```
log/
├── 20260612_091834/       ← 第1次运行
│   ├── all.log            ← 全量日志（DEBUG/INFO/WARNING/ERROR）
│   └── err.log            ← 异常日志（仅 WARNING/ERROR）
├── 20260612_104521/       ← 第2次运行
│   ├── all.log
│   └── err.log
└── 20260612_143012/       ← 最新一次运行（排错首选这里）
    ├── all.log
    └── err.log
```

**排错规则**：
- 快速定位异常 → **最新时间戳目录**的 `err.log`
- 查看完整流程 → **最新时间戳目录**的 `all.log`
- 历史对比 → 对应时间戳目录下的日志

工具函数 `get_latest_log_dir()` 返回最新日志目录路径，程序内部可直接调用。

### 日志规则

**必须做到**：
- ✅ **每个异常分支和正常分支都要写日志**，没有日志的代码不允许存在
- ✅ 日志必须写入文件（`log/<时间戳>/all.log` 和 `err.log`），不是只打印到控制台
- ✅ 每次启动都是独立的日志目录，不覆盖历史日志
- ✅ 异常日志只记录 WARNING/ERROR 级别，全量日志记录所有级别（DEBUG/INFO/WARNING/ERROR）
- ✅ 日志格式统一：`[时间] [级别] [模块] [消息]`
- ✅ 界面上的日志展示（SSE事件流等）保留，但同时写入日志文件
- ✅ 定位问题时，先看最新目录的 `err.log`，再看 `all.log`

**禁止**：
- ❌ 写没有日志的代码（任何分支都要有日志记录）
- ❌ 只打印到控制台，不写入文件
- ❌ 覆盖历史日志（每次运行都应该有独立目录）
- ❌ 依赖用户手动操作来获取错误信息
- ❌ 在多个地方分散记录日志（排错只有一处：日志文件）

### 日志格式示例

```
[2026-06-12 09:07:29] [INFO] [site_manager] 收到测试爬取请求: site=www_mimimi_top, group=视频
[2026-06-12 09:07:30] [INFO] [group_crawler] 启动子进程: pid=12345
[2026-06-12 09:07:31] [ERROR] [group_crawler] Session 创建失败: No active session 'group_crawl'
[2026-06-12 09:07:31] [INFO] [SessionGuard] 自动恢复 session...
[2026-06-12 09:07:35] [INFO] [group_crawler] 爬取完成: 20 个视频, 0 个图片
```
